📖 Overview

Ye guide aapko production-ready Docker environment setup karne mein help karegi jisme PyTorch, CUDA, GPU support, aur VS Code Dev Containers shamil hain. Iska main maqsad Reproducibility, Isolation, aur Zero-Clutter workflow provide karna hai.

🛠️ Prerequisites

1. Hardware Virtualization (BIOS/UEFI)

  • Ctrl + Shift + Esc → Performance → CPU → Check: Virtualization: Enabled
  • Agar disabled hai, BIOS mein jakar enable karein:
    • Intel: Intel Virtualization Technology (VT-x) / VT-d
    • AMD: SVM Mode / Secure Virtual Machine
  • F10 dabake Save & Exit karein.

2. WSL2 & Docker Desktop

# PowerShell (Admin)
wsl --install

Restart ke baad Ubuntu install hoga. Phir Docker Desktop install karein aur Settings → Resources → WSL Integration mein apni distro enable karein.

3. NVIDIA Drivers

nvidia-smi

Agar table aaye, drivers sahi hain. Docker automatically host driver use karega.

📄 Core Configuration Files

Dockerfile

FROM pytorch/pytorch:2.1.0-cuda11.8-cudnn8-devel

WORKDIR /workspace

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["bash"]

🔗 Volume Mounting Concept

Docker containers temporary hote hain. Data save karne ke liye Bind Mount use hota hai:

docker run --gpus all -it --rm \
  --name pytorch_container \
  -v $(pwd):/workspace \
  -w /workspace \
  pytorch_deep_project
HOST MACHINE (Windows/WSL) └── /path/to/deep_project/ ├── train.py └── data.csv ↓↓ BIND MOUNT (-v) ↓↓ CONTAINER (Linux) └── /workspace/ ├── train.py ← SAME FILE! └── data.csv ← SAME FILE!
  • Fayda 1: Host se edit → Container mein reflect
  • Fayda 2: Container delete → Code SAFE (host par hai)
  • Bina Mount: Container delete = Data LOST

💻 VS Code Dev Containers

Manual attach ki jagah, .devcontainer/devcontainer.json use karein. Ye environment ko fully automate kar deta hai:

{
  "name": "PyTorch GPU Project",
  "image": "pytorch/pytorch:2.1.0-cuda11.8-cudnn8-devel",
  "runArgs": ["--gpus", "all"],
  "workspaceMount": "source=${localWorkspaceFolder},target=/workspace,type=bind",
  "workspaceFolder": "/workspace",
  "customizations": {
    "vscode": {
      "extensions": ["ms-python.python", "ms-toolsai.jupyter"]
    }
  },
  "postCreateCommand": "pip install --no-cache-dir -r requirements.txt"
}

Usage: Folder open karo → Popup aayega → Reopen in Container dabao. Done! 🎉

🧠 Important Concepts

Concept Description Lifecycle
Image Read-only blueprint (OS + PyTorch + CUDA) Permanent (disk pe rehti hai)
Container Image ka running instance Temporary (stop/delete ho sakti hai)
Bind Mount -v host_path:/container_path Direct sync, dev ke liye best
Named Volume -v vol_name:/data Docker managed, DBs ke liye

🛠️ Troubleshooting

  • GPU nahi mil raha? → docker run --gpus all flag check karo. Docker Desktop WSL2 integration verify karo.
  • CUDA out of memory? → Batch size ghatado. WSL memory limit set karo .wslconfig file se.
  • Permission denied? → sudo usermod -aG docker $USER run karo.
  • Image purani lag rahi hai? → docker pull pytorch/pytorch:2.1.0-cuda11.8-cudnn8-devel dobara run karo.

✅ Best Practices

  • Version Pinning: requirements.txt mein exact versions likho.
  • Clean Builds: docker system prune -a regularly chalao.
  • Ignore Unnecessary: .dockerignore ko hamesha use karo.
  • Dev Containers: Team collaboration ke liye devcontainer.json commit karo.
